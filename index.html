Login and Registration for Todo App

Lab 8 Video

Description

The goal of this project was to develop a mobile task-management application (TODO app) with integrated features for user registration and authentication. React Navigation will be used for screen navigation within the application, and React Native will be used for cross-platform development. Furthermore, AsyncStorage will be used to store user login data persistently.

Skills/Technologies Used

  • - React Native: Used in the development of the mobile cross-platform application.
  • - React Navigation: Used to control screen navigation, navigating between the TODO app, the Login Screen, and the Registration Screen with the help of Stack Navigator.
  • - React Native Elements: These elements are used to create UI elements with a consistent and beautiful look, like buttons and input fields.
  • - AsyncStorage: This feature allows users to safely log in and register by permanently storing their login credentials.
  • - Regular Expressions (RegEx): Used for input validation, this tool makes sure that data entered by users corresponds to the necessary patterns for first and last names, ZIP codes, phone numbers, emails, and passwords.

Problems and Solutions

Using regular expressions to handle input validation for different user inputs, like phone numbers, emails, and passwords, was one of the challenges faced during the development process. I used a variety of RegEx patterns to evaluate each input field according to its unique requirements in order to overcome this difficulty. For instance, I used a RegEx pattern to validate phone numbers to make sure the format matches "(xxx) xxx-xxxx" and that the number is made up only of digits. Similar to this, I used a sophisticated RegEx pattern to validate emails to make sure the prefix "@" and at least one period were present. Furthermore, I used input event handlers or blur event handlers to initiate input validation as soon as the user moves away from the input field or types, giving them instant feedback on the accuracy of their inputs. In addition, I included error messages that begin with "Error:" to let users know when their inputs don't match the necessary requirements. These messages vanish as soon as the mistake is fixed. I was able to create strong input validation for the registration screen and guarantee that customers submit accurate information by skillfully utilizing regular expressions and event handling techniques.

React Native Exercise App

Lab 9 Video

Description

The goal of this project was to use React Native to create a basic mobile exercise monitoring application with distinct displays for different types of exercise. With features to raise, reset, and switch between exercises, the application lets users track both duration- and repetition-based workouts.

Skills/Technologies Used

  • - React Native: Used in the development of the mobile cross-platform application.
  • - React Navigation: This technique uses the Stack Navigator to control screen navigation.
  • - React Native Elements: Used to create UI elements with a unified and beautiful design, like buttons and lists.
  • - State Management: React component state and navigation parameters were used to manage exercise data and user interactions.

Problems and Solutions

During the development phase, one of the challenges was to ensure consistency and ease of navigation while moving workout data between screens in an efficient manner. I took a centralized approach to this by keeping a variety of exercises in the state of the Home screen component. Every piece of exercise-related data, such as its type, name, count/duration, and navigation options, is contained in this array. I made sure that every workout screen has access to the pertinent exercise data by providing this array as a parameter. I also used navigation parameters to control user inputs, such beginning the timer or increasing the count, and to dynamically show exercise specifics. Furthermore, I made use of React navigating's stack navigator feature to facilitate smooth navigating between exercises and back to the Home screen. This ensured an intuitive user experience throughout the program by enabling users to switch between workouts using both the buttons given inside each exercise screen and the back button on the device.

Quiz App


Lab 10 Video

Description

The goal of this project was to create a straightforward React Native quiz application that enables users to respond to True/False, Multiple Choice, and Multiple Answer questions. The program allows the user to navigate between questions and shows both the right answers and an overview of the user's responses.

Skills/Technologies Used

  • - React Native: Used in the development of the mobile cross-platform application.
  • - React Navigation: Used to control screen navigation, enabling users to switch between the summary and question panels.
  • - React Native Elements: Used to create UI elements with a unified and aesthetically pleasing appearance, like ButtonGroup for answering questions.
  • - State Management: React component state was used to manage the user-selected answers, score, and question index at the moment.
  • - Conditional Rendering: Logic was implemented to handle user-selected replies according to the kind of question and show various question types.
  • - Mapping and Iteration: To dynamically render each question and its associated alternatives, the list of questions was iterated over.

Problems and Solutions

Implementing the quiz's scoring mechanism and showing the right answers on the summary screen presented a problem during the development process. I took great care when creating the data structure for each question so that it would contain both the user's chosen answer and the correct answer or answers. When I got to the summary screen, I went over each question again and increased the score for each right response by comparing the user's chosen response with the correct response. Additionally, I styled the display to show whether the user's response was right or wrong by bolding the right answers and crossing out the incorrect ones. I used conditional rendering to add styles dynamically to the user's selected choices based on accuracy in order to do this. In order to compare the user-selected responses with the right ones and calculate the final score, I also employed array manipulation techniques. I overcame the difficulty and produced a workable quiz application that gives users performance feedback by carefully planning the data structure and putting in place the necessary logic for scoring and answer presentation.

Design Flash Card App

Example 9 Figma

Description

This project aims to provide a native mobile application for flashcard study. Users of the application can simply flip between cards, examine them in a random order, and create, edit, and remove flashcards. Users can also examine their progress and understand the areas where they need to concentrate their attention the most.

Skills/Technologies Used

  • - UI/UX Design: Utilized design concepts to create user interfaces that are both visually appealing and user-friendly.
  • - Figma: Created the design for the flashcard application using the collaborative interface design tool Figma.
  • - Creative Problem Solving: Overcame the challenge of designing a user interface that is both intuitive and effective for studying flashcards.

Problems and Solutions

During the design phase, ensuring that the application's interface was both user-friendly and functional presented a challenge. To address this, I conducted thorough research into existing flashcard applications to identify common design patterns and user interface elements. I then created wireframes and prototypes in Figma, iterating on the designs based on feedback from peers and usability testing. By carefully considering user needs and incorporating best practices in UI/UX design, I was able to create an intuitive and visually appealing interface for the flashcard application.
